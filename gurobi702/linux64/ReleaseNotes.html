Release Notes for Version 7.0

In addition to many performance enhancements, Gurobi 7.0 adds several new features:

  • Python Modeling Enhancements: Our Python modeling interface has been greatly enhanced. We provide new methods and classes that further simplify the task of translating mathematical models into efficient implementations. For an introduction see the Python API Overview section in the Reference Manual.
  • Multiple Objective Support: You can now associate multiple, independent optimization objectives with your model, and then perform either blended or lexicographic optimization on the resulting multi-objective model. For details see the multiple objective section in the Reference Manual.
  • MIP Solution Pool Support: You can now obtain more than just one optimal solution to your MIP model. Specifically, you can now request the N best solutions, or N solutions that are less than a specified gap from the optimal solution. For details see the solution pool section in the Reference Manual.
  • General constraints: You can now enter several commonly occuring types of constraints without having to translate them into linear constraints yourself. Specifically, you can now add the following constraint types:
    • MIN/MAX constraint: sets a decision variable equal to the min/max over a set of other decision variables
    • ABS constraint: sets a decision variable equal to the absolute value of some other decision variable
    • AND/OR constraint: sets a binary decision variable equal to the logical and/or over a set of other binary decision variables
    • INDICATOR constraint: forces a linear constraint to be satisfied whenever a binary decision variable takes the specified value
    For details see the General Constraints discussion in the variables and constraints section in the Reference Manual.
  • No more update calls for most programs: The new release changes the default update behavior. You can now refer to newly created variables and constraints without having to call update first. You can set the UpdateMode parameter to 0 if you want to revert to the previous behavior.
  • Tuning tool improvements: The tuning tool has been improved. It now accepts MIP starts. Also, in previous releases the tuning tool would always try to find settings that minimized the optimality gap. You can now use the new TuneCriterion parameter to search for settings that produce the best lower bound, upper bound, or gap.
  • Enhanced Gurobi Instant Cloud: You can now launch Instant Cloud instances directly from the Gurobi APIs (or through a license file).
  • Enhanced .NET property support: Parameters and attributes are now available through .NET properties in the .NET interface.
  • Simplified parameter setting: You now set parameters on the model object itself in the object-oriented interfaces, which removes a frequent source of confusion.
  • Support for Python 3.5 on Mac: We've added support for Python 3.5 (including Anaconda 3.5) on Mac.
  • Additional new parameters:
    • BestObjStop: New termination criterion - terminates a MIP solve when the incumbent objective is at least as good as the specified value.
    • BestBdStop: New termination criterion - terminates a MIP solve when the best bound is at least as good as the specified value.
    • InfProofCuts: New cutting plane type.
    • StrongCGCuts: New cutting plane type.
    • DegenMoves: Allows the user to shut off a root heuristic that can sometimes be expensive.

Code Conversion from Version 6.5

Programs that were originally written for version 6.5 of the Gurobi Optimizer generally won't require any code changes to use version 7.0. If you'd like to use the new features, some code changes are required.
